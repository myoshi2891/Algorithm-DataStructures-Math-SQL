function getPermutation(n: number, k: number): string {
    // 階乗を事前計算
    const factorial: number[] = [1];
    for (let i = 1; i < n; i++) {
        factorial[i] = factorial[i - 1] * i;
    }
    
    // 使用可能な数字のリスト
    const numbers: number[] = [];
    for (let i = 1; i <= n; i++) {
        numbers.push(i);
    }
    
    // k を 0-indexed に変換
    k--;
    
    let result: string = '';
    
    // 各桁を順番に決定
    for (let i = 0; i < n; i++) {
        // インデックス計算
        const index: number = Math.floor(k / factorial[n - 1 - i]);
        
        // 数字を結果に追加
        result += numbers[index];
        
        // 使用済み数字を削除
        numbers.splice(index, 1);
        
        // k を更新
        k %= factorial[n - 1 - i];
    }
    
    return result;
}

アルゴリズム概要

核心アイデア

全順列を生成せず、数学的計算で直接k番目の順列を求める

キーポイント

  • • 階乗による位置計算
  • • 各桁での数字選択
  • • 使用済み数字の除去
  • • 剰余演算での位置更新

効率性

O(n!)の全順列生成を回避し、O(n²)で直接計算

時間計算量解析

階乗計算

O(n) - n個の階乗を順次計算

for i = 1 to n-1: factorial[i] = factorial[i-1] * i

メインループ

O(n²) - n回のループ、各回でsplice操作O(n)

n × O(n) = O(n²)

総合計算量

O(n) + O(n²) = O(n²)

結論: O(n²) vs 全順列生成 O(n! × n)

空間計算量解析

factorial配列

O(n) - n個の階乗値を保存

配列サイズ: n

numbers配列

O(n) - 使用可能な数字を保存

初期サイズ: n → 最終サイズ: 0

その他変数

O(1) - 定数個の変数

総空間計算量: O(n)
